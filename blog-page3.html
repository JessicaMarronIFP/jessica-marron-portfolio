En este proyecto reproduje y adapté un notebook de Kaggle para realizar análisis de sentimientos sobre tweets (positivo/negativo) utilizando el dataset tipo Sentiment140. El trabajo se centró en comprender el flujo completo de un proyecto de PLN, ejecutándolo en local y optimizándolo para entrenar en CPU mediante muestreo del dataset y ajustes de hiperparámetros.

Implementé un preprocesado avanzado del texto, incluyendo normalización de URLs, menciones y hashtags, tratamiento de emoticonos, y conservación de señales relevantes para sentimiento (negaciones, intensificadores y puntuación expresiva). Además, generé características manuales (conteos y ratios de palabras positivas/negativas, emoticonos, exclamaciones, etc.) y las normalicé con StandardScaler para combinarlas con el modelo.

Python · Pandas · NumPy · Scikit-learn · TensorFlow/Keras · Kaggle/Jupyter Notebook

El modelo final es híbrido: una rama de LSTM bidireccional con embeddings para capturar el contexto del texto y otra rama con features numéricas, fusionadas mediante concatenación y capas densas con regularización (dropout, spatial dropout, early stopping y reduce LR). Con una muestra de ~30.000 tweets y 3 épocas de entrenamiento, el modelo alcanzó un rendimiento de Accuracy 0.766, F1 0.755 y AUC-ROC 0.842, analizando también la matriz de confusión para interpretar errores y sesgos del modelo.
